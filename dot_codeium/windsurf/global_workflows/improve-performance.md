ปรับปรุงประสิทธิภาพที่ไม่กระทบ ui ที่ทำให้สวยน้อยลง

0. /analyze-project

1. **ปรับปรุงประสิทธิภาพ Rendering**
   - **ลดความซับซ้อน (Reduce Complexity):** ลดจำนวนองค์ประกอบที่ต้องแสดงผล หรือลดความซับซ้อนของแต่ละองค์ประกอบ
   - **Batching/Grouping:** รวมการอัปเดต UI หลายๆ ครั้งให้เป็นการอัปเดตครั้งเดียว เพื่อลดการ re-render ที่ไม่จำเป็น
   - **Lazy Loading:** โหลดเฉพาะสิ่งที่ผู้ใช้เห็นในหน้าจอเท่านั้น ส่วนที่เหลือค่อยโหลดเมื่อผู้ใช้เลื่อนไปถึง

2. **ปรับปรุงประสิทธิภาพ Startup**
   - **ลดงานตอนเริ่มต้น (Defer Non-critical Work):** ทำงานเฉพาะที่จำเป็นจริงๆ ในตอนเริ่มต้นโปรแกรม ส่วนที่เหลือให้ทำเมื่อถูกเรียกใช้งาน
   - **Code Splitting:** แบ่งโค้ดเป็นส่วนเล็กๆ และโหลดเฉพาะส่วนที่จำเป็นต่อการทำงานเริ่มต้น
   - **Pre-computation/Caching:** คำนวณผลลัพธ์ที่ใช้บ่อยๆ ไว้ล่วงหน้าและเก็บไว้ เพื่อลดเวลาในการคำนวณตอนเริ่มต้น

3. **ปรับปรุงประสิทธิภาพ Network**
   - **ลดขนาดข้อมูล (Payload Size Reduction):** บีบอัดข้อมูล (เช่น Gzip) และใช้รูปแบบข้อมูลที่มีขนาดเล็ก (เช่น Protocol Buffers แทน JSON)
   - **ลดจำนวน Request (Request Batching):** รวมหลายๆ request เป็นอันเดียว
   - **Caching:** เก็บข้อมูลที่เรียกใช้บ่อยๆ ไว้ที่ client หรือ server เพื่อลดการเรียกซ้ำ

4. **ปรับปรุงประสิทธิภาพ I/O (Input/Output)**
   - **Buffering:** อ่านหรือเขียนข้อมูลเป็นก้อนใหญ่ๆ แทนที่จะทำทีละเล็กทีละน้อย เพื่อลด overhead ของ I/O operations
   - **Asynchronous I/O:** ใช้ non-blocking I/O เพื่อให้โปรแกรมสามารถทำงานอื่นต่อไปได้ระหว่างรอ I/O
   - **เลือกใช้ Storage ที่เหมาะสม:** ใช้หน่วยความจำที่เร็วขึ้น (เช่น SSD แทน HDD) สำหรับข้อมูลที่เข้าถึงบ่อย

5. **ปรับปรุงประสิทธิภาพ Memory**
   - **ลดการสร้าง Object (Object Pooling):** นำ Object ที่มีอยู่แล้วกลับมาใช้ใหม่แทนการสร้างใหม่ทุกครั้ง
   - **จัดการ Memory อย่างมีประสิทธิภาพ:** คืน Memory ที่ไม่ใช้แล้วทันที หรือหลีกเลี่ยง Memory Leaks
   - **ใช้โครงสร้างข้อมูลที่เหมาะสม:** เลือกโครงสร้างข้อมูลที่ใช้หน่วยความจำน้อยที่สุดสำหรับงานนั้นๆ

6. **ปรับปรุงประสิทธิภาพ Bundle Size**
   - **Tree Shaking/Dead Code Elimination:** ลบโค้ดที่ไม่ได้ใช้งานออกจาก final bundle
   - **Minification:** ลดขนาดโค้ดโดยการลบช่องว่าง, comment และย่อชื่อตัวแปร
   - **ใช้ Library ขนาดเล็ก:** เลือกใช้ Library ที่มีขนาดเล็กและทำงานได้เหมือนกัน

7. **ปรับปรุงประสิทธิภาพ Concurrency**
   - **Parallelism:** แบ่งงานใหญ่เป็นงานย่อยๆ แล้วทำงานพร้อมกันบนหลายๆ CPU core
   - **หลีกเลี่ยง Race Conditions:** ใช้ locks, mutexes, หรือ data structure ที่เป็น thread-safe เพื่อป้องกันปัญหาข้อมูลไม่ตรงกัน
   - **Non-blocking Algorithms:** ใช้วิธีการที่ไม่ต้องรอ lock เพื่อเพิ่มประสิทธิภาพในการทำงานพร้อมกัน

8. **ปรับปรุงประสิทธิภาพ Throughput**
   - **เพิ่มประสิทธิภาพ Algorithm:** ใช้ Algorithm ที่มี Big O notation ที่ดีกว่า (เช่น O(log n) แทน O(n))
   - **Load Balancing:** กระจายงานไปยังหลายๆ server หรือ process เพื่อให้ทำงานได้มากขึ้น
   - **Pipelining:** แบ่งขั้นตอนการทำงานและส่งผลลัพธ์ของขั้นตอนหนึ่งไปยังอีกขั้นตอนหนึ่งทันที โดยไม่ต้องรอให้งานทั้งหมดเสร็จ
